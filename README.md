# novpn

По состоянию репозитория это статическое web-приложение для подготовки и редактирования `amnezia_sites.json` для split-tunnel в AmneziaVPN. Runtime у проекта простой: вся рабочая часть находится в `public/`, backend и база данных не используются, сборки фронтенда нет. Приложение загружает встроенный каталог сервисов из `public/services.json`, даёт редактировать правила построчно, проверяет формат строк, дорезолвливает домены через DoH и скачивает итоговый JSON.

Сейчас в каталоге `public/services.json` лежат 23 готовых сервиса в 2 категориях: 15 российских и 8 зарубежных. Вспомогательные Python-скрипты в `scripts/` нужны только для обслуживания каталога и не участвуют в обычном запуске приложения.

## Что умеет приложение

- Загружать существующий JSON через кнопку или drag-and-drop.
- Показывать редактируемый построчный список и итоговый JSON рядом.
- Подсвечивать некорректные строки и блокировать скачивание при ошибках.
- Автоматически преобразовывать одиночный IP `1.2.3.4` в `1.2.3.4/32`.
- Резолвить домены без IP через DoH: сначала Cloudflare, затем Google.
- Добавлять готовые наборы правил из встроенного каталога сервисов.
- Скачивать результат как `amnezia_sites.json`.

## Важные особенности

- Приложение нужно открывать через HTTP/HTTPS. Прямой запуск `public/index.html` через `file://` не подходит, потому что интерфейс читает `services.json` через `fetch()`.
- Для автоматического резолва доменов браузеру нужен доступ к `cloudflare-dns.com` и `dns.google`.
- Комментарии вида `# comment` в редакторе поддерживаются и сохраняются в экспортируемом JSON через служебные записи.

## Структура проекта

- `public/index.html` — HTML интерфейса.
- `public/app.js` — основная логика клиента: загрузка файла, валидация, DoH-резолв, экспорт JSON, модалка сервисов.
- `public/styles.css` — стили интерфейса.
- `public/services.json` — встроенный каталог сервисов.
- `Dockerfile` — контейнер со статической раздачей через `nginx:alpine`.
- `docker-compose.yml` — готовый сценарий запуска контейнера на порту `8099`.
- `scripts/verify_services.py` — служебная проверка и обновление части записей в `public/services.json` через DoH и RIPEstat.
- `scripts/build_services.py` — служебная пересборка `public/services.json`; для зарубежных сервисов ожидает локальную папку `bats/`, которой нет в опубликованном репозитории.

## Формат строк в редакторе

- `10.0.0.0/8` — сеть в CIDR.
- `1.2.3.4` — одиночный IP; при проверке будет преобразован в `1.2.3.4/32`.
- `example.com 1.2.3.4` — домен с фиксированным IP.
- `example.com` — домен без IP; адрес будет дорезолвен при нажатии `Проверить`.
- `# Comment` — комментарий или разделитель.

## Развертывание

### 0. Общий шаг для всех сценариев

Для новой папки нужен `git clone`, а не `git pull`:

```bash
git clone https://github.com/poznik/novpn.git
cd novpn
```

### 1. Запуск на железе

Подходит любой статический HTTP-сервер. Самый простой вариант — через Python 3:

```bash
python -m http.server 8099 --directory public
```

Если в системе команда `python` не настроена, используйте `python3`.

После запуска приложение будет доступно по адресу:

```text
http://localhost:8099
```

Остановить сервер:

```text
Ctrl+C
```

### 2. Запуск в контейнере

Требуется Docker с `docker compose`.

```bash
docker compose up -d --build
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:8099
```

Полезные команды:

```bash
docker compose logs -f
docker compose down
```

## Служебные сценарии

Обычному запуску приложения эти скрипты не нужны, но они полезны для поддержки каталога сервисов.

Проверка и обновление `public/services.json`:

```bash
python scripts/verify_services.py
```

Пересборка `public/services.json`:

```bash
python scripts/build_services.py
```

Ограничения:

- `verify_services.py` требует сетевого доступа к DoH и RIPEstat.
- `build_services.py` использует локальные `.bat`-источники из папки `bats/`; в опубликованном репозитории этой папки нет.
